Semantic status colors (success/warning/danger) are intentionally separate from the brand accent — a job match score and a "profile field needs verification" warning should never fight for the same visual weight.

Part F — Mobile & Accessibility

Small screens, and everyone

Mobile components

Same nav-tabbar shown above is the mobile primary nav. Job cards and the chat panel both collapse to full device width with no layout change beyond that — no separate "mobile version" of any component to maintain.

Accessibility states

  • Accent tokens are tuned per mode specifically to clear WCAG AA text contrast — not the same hex reused with a filter
  • Status chips carry a text label, never color alone — verified in the specimen above
  • The one animation in this system (coach "thinking") degrades to a static state under prefers-reduced-motion
